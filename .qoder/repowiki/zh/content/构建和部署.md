## 开发环境搭建

### 系统要求
- **Go版本**: 1.21+ (项目使用1.26.1)
- **Node.js版本**: 16.x+ (推荐20.12.2，兼容glibc 2.28+)
- **编译工具**: gcc/g++ (SQLite CGO编译依赖)
- **数据库**: SQLite (内置支持)
- **JMeter**: 5.6.3+ (Java 11+运行时)

### 环境准备步骤
1. **安装Go环境**
   ```bash
   # 检查Go版本
   go version
   
   # 设置GOPATH和PATH
   export GOPATH=$HOME/go
   export PATH=$PATH:$GOPATH/bin
   ```

2. **安装Node.js和npm**
   ```bash
   # 检查Node.js版本
   node --version
   
   # 配置npm镜像源
   npm config set registry https://registry.npmmirror.com
   ```

3. **安装编译工具**
   ```bash
   # Ubuntu/Debian
   sudo apt-get update
   sudo apt-get install gcc g++ build-essential
   
   # CentOS/RHEL/Fedora
   sudo yum install gcc gcc-c++ make
   sudo dnf install gcc gcc-c++ make
   ```

4. **验证JMeter环境**
   ```bash
   # 检查Java版本
   java -version
   
   # 验证JMeter安装
   jmeter --version
   ```

## 构建系统详解

### Makefile构建命令体系

#### 基础构建命令
- **build-frontend**: 安装Node.js依赖并执行Vite构建，生成web/dist目录
- **build-backend**: 启用CGO编译后端，生成单文件可执行文件
- **build-all**: 先构建前端再构建后端，完整构建流程
- **build-linux**: 交叉编译Linux版本，设置GOOS=linux GOARCH=amd64

#### 开发模式命令
- **dev**: 同时启动后端Go开发服务器和前端Vite开发服务器
- **dev-backend**: 仅启动后端Go开发服务器
- **dev-frontend**: 仅启动前端Vite开发服务器，支持端口环境变量

#### 清理命令
- **clean**: 清理可执行文件和web/dist构建产物

### 构建优化策略

#### CGO编译配置
```makefile
# 启用CGO以支持SQLite驱动
CGO_ENABLED=1 go build -o jmeter-admin .

# 交叉编译Linux版本
CGO_ENABLED=1 GOOS=linux GOARCH=amd64 go build -o jmeter-admin .
```

#### 构建性能优化
- 使用并行构建减少等待时间
- 缓存Node.js依赖避免重复安装
- 交叉编译支持多平台部署

#### 开发服务器配置
```javascript
// Vite开发服务器
server: {
  port: frontendPort,
  proxy: {
    '/api': {

      rewrite: (path) => path.replace(/^\/api/, '/api')
```

## 部署策略

### 一键部署（Linux服务器）

#### 快速部署流程
```bash
# 安装所有依赖（Go、Node.js、gcc、Java、JMeter）
./deploy.sh install-deps
source ~/.bashrc

# 编译项目
./deploy.sh install

# 启动服务
./deploy.sh start

# 访问 http://your-server-ip:8080
```

#### 一键部署脚本功能
- **install-deps**: 自动安装所有依赖组件
- **install**: 编译项目（前端+后端）
- **start/stop/restart/status**: 服务管理
- **install-service**: 安装systemd服务

### 本地开发模式

#### 开发环境启动
```bash
# 同时启动前后端（前端热更新）
make dev

# 或分别启动
make dev-backend    # 后端 :8080
make dev-frontend   # 前端 :3000（代理 API 到 :8080）
```

#### 开发模式特性
- 前端热更新支持
- API代理到后端
- 端口可配置环境变量

### 编译部署选项

#### 编译命令
```bash
# 完整编译（前端 + 后端）
make build-all

# 仅编译后端（需先构建前端）
make build-backend

# 交叉编译 Linux 版本
make build-linux

# 运行
./jmeter-admin
```

## 生产环境配置优化

### 网络与端口安全

#### 端口配置
- **HTTP服务**: 8080 (可配置)
- **JMeter端口**: 50000 (RMI端口)
- **RMI端口**: 1099 (RMI注册中心)

#### 防火墙配置
```bash
# 仅开放必需端口
sudo ufw allow 8080/tcp
sudo ufw allow 50000/tcp
sudo ufw allow 1099/tcp

# 限制来源IP
sudo ufw allow from 192.168.1.0/24 to any port 8080
```

### 配置文件管理

#### 安全配置
```yaml
# 生产环境配置示例
server:
  port: 8080

jmeter:
  path: "/opt/apache-jmeter-5.6.3/bin/jmeter"
  master_hostname: "192.168.62.225"

dirs:
  data: "/var/lib/jmeter-admin/data"
  uploads: "/var/lib/jmeter-admin/uploads"
  results: "/var/lib/jmeter-admin/results"
```

#### 权限控制
```bash
# 设置正确的文件权限
sudo chown -R www-data:www-data /var/lib/jmeter-admin
sudo chmod -R 755 /var/lib/jmeter-admin
```

### 性能优化配置

#### JVM内存配置
```bash
# 通过JVM_ARGS环境变量自定义
export JVM_ARGS="-Xms2g -Xmx8g -XX:+UseG1GC"
```

#### 数据库优化
```sql
-- SQLite性能优化设置
```

## 故障排除指南

### 编译相关问题

#### CGO编译错误
```bash
# 错误: cannot find -lsqlite3
sudo apt-get install libsqlite3-dev

# 错误: gcc not found
sudo apt-get install build-essential

# 错误: CGO disabled
export CGO_ENABLED=1
```

#### Go模块下载失败
```bash
# 设置Go代理
go env -w GOPROXY=https://goproxy.cn,direct
go env -w GOSUMDB=sum.golang.google.cn

# 清理缓存
go clean -cache -modcache
```

### 前端构建问题

#### Node.js版本不兼容
```bash
# 检查glibc版本
ldd --version

# 选择兼容的Node.js版本
# glibc >= 2.28: Node.js 20.x
# glibc < 2.28: Node.js 16.x
```

#### 依赖安装失败
```bash
# 使用国内镜像源
npm config set registry https://registry.npmmirror.com

# 清理缓存
rm -rf node_modules package-lock.json
npm ci
```

### 运行时问题

#### 端口占用
```bash
# 查找占用端口的进程
lsof -i :8080
netstat -tulpn | grep :8080

# 杀死进程
kill -9 $(lsof -t -i :8080)
```

#### 权限问题
```bash
# 设置正确权限
chmod +x jmeter-admin
chown -R www-data:www-data /var/lib/jmeter-admin

# 检查SELinux
setenforce 0  # 临时禁用
```

#### 数据库问题
```bash
# 检查数据库文件
ls -la data/jmeter-admin.db

# 修复损坏的数据库
sqlite3 data/jmeter-admin.db ".recover" > recovered.sql
```

### 网络连接问题

#### JMeter连接失败
