## 简介
本指南提供了JMeter Admin项目的完整构建和部署解决方案，涵盖开发环境搭建、构建系统配置、前端Vite构建流程、后端编译与CGO依赖处理、多种部署策略（单文件、容器化、传统）、生产环境配置优化与安全加固、自动化部署脚本使用与定制，以及部署后的监控、日志与故障排除建议。

## 开发环境搭建

### 系统要求
- **Go版本**: 1.21+ (项目使用1.26.1)
- **Node.js版本**: 18+ (推荐20.12.2，兼容glibc 2.28+)
- **编译工具**: gcc/g++ (SQLite CGO编译依赖)
- **数据库**: SQLite (内置支持)
- **JMeter**: 5.6.3+ (Java 11+运行时)

### 环境准备步骤
1. **安装Go环境**
   ```bash
   # 检查Go版本
   go version
   
   # 设置GOPATH和PATH
   export GOPATH=$HOME/go
   export PATH=$PATH:$GOPATH/bin
   ```

2. **安装Node.js和npm**
   ```bash
   # 检查Node.js版本
   node --version
   
   # 配置npm镜像源
   npm config set registry https://registry.npmmirror.com
   ```

3. **安装编译工具**
   ```bash
   # Ubuntu/Debian
   sudo apt-get update
   sudo apt-get install gcc g++ build-essential
   
   # CentOS/RHEL/Fedora
   sudo yum install gcc gcc-c++ make
   sudo dnf install gcc gcc-c++ make
   ```

4. **验证JMeter环境**
   ```bash
   # 检查Java版本
   java -version
   
   # 验证JMeter安装
   jmeter --version
   ```

## 构建系统详解

### Makefile构建命令体系

#### 基础构建命令
- **build-frontend**: 安装Node.js依赖并执行Vite构建，生成web/dist目录
- **build-backend**: 启用CGO编译后端，生成单文件可执行文件
- **build-all**: 先构建前端再构建后端，完整构建流程
- **build-linux**: 交叉编译Linux版本，设置GOOS=linux GOARCH=amd64

#### 开发模式命令
- **dev**: 同时启动后端Go开发服务器和前端Vite开发服务器
- **dev-backend**: 仅启动后端Go开发服务器
- **dev-frontend**: 仅启动前端Vite开发服务器，支持端口环境变量

#### 清理命令
- **clean**: 清理可执行文件和web/dist构建产物

### 构建优化策略

#### CGO编译配置
```makefile
# 启用CGO以支持SQLite驱动
CGO_ENABLED=1 go build -o jmeter-admin .

# 交叉编译Linux版本
CGO_ENABLED=1 GOOS=linux GOARCH=amd64 go build -o jmeter-admin .
```

#### 构建性能优化
- 使用并行构建减少等待时间
- 缓存Node.js依赖避免重复安装
- 交叉编译支持多平台部署

## 多种部署策略

### 单文件部署

#### 部署流程
1. **编译应用**
   ```bash
   make build-all
   ```

2. **创建配置文件**
   ```bash
   cp config.yaml.example config.yaml
   ```

3. **启动服务**
   ```bash
   ./jmeter-admin
   ```

#### 优势特点
- 单个可执行文件，便于分发
- 无外部依赖，部署简单
- 内置前端资源，无需额外服务器

### 容器化部署

#### Docker构建
```dockerfile
FROM alpine:latest

# 安装运行时依赖
RUN apk add --no-cache ca-certificates

# 复制可执行文件
COPY jmeter-admin /usr/local/bin/jmeter-admin

# 创建数据目录
RUN mkdir -p /app/data /app/uploads /app/results

# 暴露端口
EXPOSE 8080

# 启动命令
CMD ["/usr/local/bin/jmeter-admin"]
```

#### Docker Compose配置
```yaml
version: '3.8'
services:
  jmeter-admin:
    build: .
    ports:
      - "8080:8080"
    volumes:
      - ./data:/app/data
      - ./uploads:/app/uploads
      - ./results:/app/results
    environment:
      - BACKEND_PORT=8080
      - FRONTEND_PORT=3000
```

### 传统部署

#### 一键部署脚本
```bash
# 安装所有依赖
./deploy.sh install-deps

# 编译项目
./deploy.sh install

# 启动服务
./deploy.sh start

# 查看状态
./deploy.sh status
```

- **JMeter端口**: 50000 (RMI端口)

```bash
# 仅开放必需端口
```

## 自动化部署与CI/CD

### GitHub Actions配置

#### CI/CD工作流
```yaml
name: Build and Deploy

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  build:
    runs-on: ubuntu-latest
    
    steps:
    - uses: actions/checkout@v4
    
    - name: Set up Go
      uses: actions/setup-go@v4
      with:
        go-version: '1.26.1'
    
    - name: Setup Node.js
      uses: actions/setup-node@v3
      with:
        node-version: '20'
    
    - name: Install dependencies
      run: |
        cd web
        npm ci
    
    - name: Build frontend
      run: |
        cd web
        npm run build
    
    - name: Build backend
      run: |
        go build -o jmeter-admin .
    
    - name: Upload artifacts
      uses: actions/upload-artifact@v3
      with:
        name: jmeter-admin
        path: jmeter-admin
```

### Jenkins集成

#### Jenkins Pipeline
```groovy
pipeline {
    agent any
    
    stages {
        stage('Checkout') {
            steps {
                git branch: 'main'
            }
        }
        
        stage('Setup Environment') {
            steps {
                sh 'go version'
                sh 'node --version'
            }
        }
        
        stage('Build Frontend') {
            steps {
                sh 'cd web && npm ci && npm run build'
            }
        }
        
        stage('Build Backend') {
            steps {
                sh 'go build -o jmeter-admin .'
            }
        }
        
        stage('Deploy') {
            steps {
                sh './deploy.sh install'
                sh './deploy.sh start'
            }
        }
    }
}
```

### Docker持续部署

#### 多阶段构建
```dockerfile
# 构建阶段
FROM golang:1.26.1-alpine AS builder

WORKDIR /app
COPY . .

# 安装编译工具
RUN apk add --no-cache gcc g++ make

# 构建后端
RUN CGO_ENABLED=1 go build -o jmeter-admin .

# 前端构建
FROM node:20-alpine AS frontend-builder
WORKDIR /app
COPY web/ web/
RUN npm ci && npm run build

# 最终镜像
FROM alpine:latest

RUN apk add --no-cache ca-certificates

# 复制构建产物
COPY --from=builder /app/jmeter-admin /usr/local/bin/jmeter-admin
COPY --from=frontend-builder /app/web/dist /app/web/dist

EXPOSE 8080
CMD ["/usr/local/bin/jmeter-admin"]
```
